# Day 1 - Microsoft Intune Fundamentals

## What is Microsoft Intune?

Microsoft Intune is a cloud-based endpoint management solution that helps organizations manage, secure, and monitor devices remotely.

### Example

Instead of configuring 200 laptops one by one, an administrator creates one policy in Intune, and all enrolled devices receive it automatically.

---

## What is an Endpoint?

An endpoint is any device used by an employee.

Examples:

- Windows Laptop
- Desktop
- Android Phone
- iPhone
- Tablet

---

## What is MDM?

MDM (Mobile Device Management) manages the entire device.

It allows administrators to:

- Install applications
- Configure settings
- Lock devices
- Wipe devices
- Enable BitLocker

### Example

A company-owned Windows laptop.

---

## What is MAM?

MAM (Mobile Application Management) manages only company applications.

### Example

Your personal phone has Outlook and Teams installed.

If you leave the company, only company data is removed while your personal photos and apps remain.

---

## MDM vs MAM

| MDM | MAM |
|------|------|
| Manages the whole device | Manages only company apps |
| Used mainly for company-owned devices | Used mainly for personal devices |
| Full device control | App-level control |

---

## What Does an Intune Administrator Do?

- Enroll devices
- Deploy applications
- Create security policies
- Configure BitLocker
- Manage Windows Updates
- Monitor compliance
- Troubleshoot devices

---

## Key Terms

| Term | Meaning |
|------|---------|
| Endpoint | Company-managed device |
| Policy | Rules created by an administrator |
| Enrollment | Registering a device into Intune |
| Compliance | Device follows company security rules |
| Cloud | Services managed over the internet |

---

## Real-Life Example

Think of a school.

Teacher = Intune Administrator

Students = Devices

School Rules = Policies

The teacher creates one rule, and every student follows it.

Similarly, an Intune Administrator creates one policy, and every enrolled device receives it automatically.
